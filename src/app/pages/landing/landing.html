The Approach

Three ways to build a design system.
Only one survives at scale.

Not a UI kit. Not a utility framework. Not Tailwind or Bootstrap. A design architecture built to last.

Option A · UI Kits

Pre-built components

Fast to start, impossible to maintain. Locked into a vendor's visual language and update cadence.

Atomic utilities

Foundations

Built on principles,
not just syntax.

data_object

Atomic Design Tokens

From base values to semantic aliases. A hierarchy that ensures visual consistency across all components.

layers

CSS Cascade Layers

Harnessing @layer to solve specificity wars forever. Reset, Base, Plugins, Components, and Utilities separated by design.

straighten

REM-based Sizing

Accessibility is not optional. StyleBase enforces REM for spacing and sizing. Borders use px; media queries are always written in px.

contrast

Theme-Aware by Default

Light and dark themes auto-switch via prefers-color-scheme. A data-theme attribute always wins.

Architecture

Structured for the long game.

A battle-tested directory structure that separates concerns and prevents the "CSS spaghetti" effect in large projects.

folder tokens/ Colors, Font, Spacing, Themes
folder abstracts/ Mixins, Functions, Tools
folder components/ Button, Badge, Textfield…
folder utilities/ Flex, Grid, Spacing, Display
Ships with stylelint prettier husky lint-staged
main.scss 
// Base
@include meta.load-css('base/reset');

// Tokens
@include meta.load-css('tokens/colors');
@include meta.load-css('tokens/themes/light');

// Components
@include meta.load-css('components/button');
@include meta.load-css('components/badge');

// Utilities
@include meta.load-css('utilities');
Quick Start

Up and running in three steps.

01

Copy the scaffold

Drop the src/scss/ directory into your project. No install, no CLI, no lock-in.

cp -r stylebase/src/scss ./src
02

Configure your tokens

Edit the token files to match your brand - colors, type scale, spacing. One change propagates everywhere.

code src/scss/tokens/_colors.scss
03

Import and build

Import main.scss in your app and start writing semantic, scalable CSS.

@use 'scss/main' as *;
